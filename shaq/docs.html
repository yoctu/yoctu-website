Introduction

Shaq can be used in many ways by your TMS or ERP.

It is a full open bidding system where you can define your own rules. There is almost no limit on the possibilities you have.

It is composed of 3 major components : an Api, a Websocket and a Web Application.

The minimum you can get is the only Api version. You can then create or query your shaqs, bids, chats or any other information the way you like/need it. But you need to understand there is a restriction : a rate limiting has been setup on the Api: As default you have 20 requests/min for 1000 shaqs/month and below. You get an extra 1 requests/min every 100 shaqs/months.

That is why we also recommend to use the websocket service. There is no limit, you get realtime event and you can easily connect your application to it.

if you do not have an application, you can then use the default web application provided by shaq. You can start shaqqing from tomorrow. It has all the features and can be customized. But customization has also some limits. If you need a complete different look and fell or want to define your own rules you need then to create yours !
