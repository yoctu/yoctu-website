Shaq Entity Description: 


		id : Shaq unique id. If not provided it is generated by backend server.
		name : Shaq label. Used for display only.
		valid_from : Shaq decision date/time. Before that time no accept/decline is possible. valid from must be greater than NOW and lower than valid_until.
		valid_until : Shaq closing date/time. Shaq will be closed on that date/time. valid_until must be lower than puDate.
		reported_at : Shaq creation date/time. It should be NOW by default.
		key : Key used to match and create relation with bids, chats, notifications or any other entities. it must be unique.
		source : list of creator id (Autioneers). It is used for Authorisation and Acls. if a usercode is part of the list it then considered as owner of the sha.
		sourceName : Name for the owner ID(s). It is a label used for display. Should match order and number from source attribute.
		sourceEmail : Email for the owner ID(s). It is used to send emails or give this information to the bidders. It also should match order in source attribute.
		target : List of bidder IDs. used for Authorisation and Acls. if a usercode is part of the list it then considered as invited or approved bidder
		targetName : Name for the bidder IDs. It is used for display. It must match the order in target attribute.
		targetEmail : Email for the bidder IDs. It is used to send emails or provide information to shaq owner. It must match order in target list.
		targetStatus : Status of the bidders. It can be : empty, Removed, Giveup.
		type : it is public or private. A private shaq is only visible to invited bidders from the shaq owner. Public is accessible to anyone and they can subscribe to it for bidding.
		status : Shaq status. It can have 4 statuses : running, expired, cancelled, completed. They are self explained.
		puPlace : Define the pickup place of the package. You must provide 5 fields : Address, Post code, City, Country, Country Code.
		dePlace : Define the delivery place of the package. You must provide 5 fields : Address, Post code, City, Country, Country Code.
		puContact : Contact of the person on the pickup place. You must provide 4 fields : company name, contact name, email and  phone.
		deContact : Contact of the person on the delivery place. You must provide 4 fields : company name, contact name, email and  phone.
		distance : distance in kilometer from pickup and delivery.
		dimension : package information. You must provide 6 informations : number of package of this dimension, width, length, height, weight, note/description. you can provide multiple dimension if you have multiple package type.

for the owner(s) all the information are sent.
for the bidders we hide all the targets information (target, targetEmail, targetStatus, targetName) except himself and the bidprice information.
